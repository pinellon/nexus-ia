# Nexus AI v0.1.1 Coder Tools Plan

## Objective

Nexus AI v0.1.1 adds deterministic programming-agent utilities on top of the v0.1 controlled local beta. These tools help inspect a repository, select context, plan a programming task, suggest tests, and review risk without applying patches automatically.

## Functionality

- `NexusAI/repo_indexer.py`: indexes text/code files, ignores generated and sensitive directories, detects probable file types, and emits summaries.
- `NexusAI/context_builder.py`: selects relevant files for a task using path, keyword, extension, type, test, docs, and config signals.
- `NexusAI/planner.py`: returns a deterministic structured plan with the steps `understand_task`, `locate_relevant_files`, `propose_change`, `suggest_tests`, and `assess_risk`.
- `NexusAI/diff_utils.py`: creates safe diff representations and detects blocked paths or dangerous content without applying changes.
- `NexusAI/code_reviewer.py`: reviews selected files or proposed changes for sensitive paths, dangerous commands, missing tests, large changes, and scope risk.
- `NexusAI/test_suggester.py`: suggests Python and Node/TypeScript validation commands without running them.
- `NexusAI/coder_agent.py`: orchestrates indexing, context selection, planning, test suggestion, and risk review into one JSON report.

## CLI Commands

```powershell
python NexusAI/repo_mode.py index --root .
python NexusAI/repo_mode.py plan --task "add validation for dangerous commands"
python NexusAI/repo_mode.py coder-task --task "review project structure and suggest tests"
```

Existing v0.1 commands remain available:

```powershell
python NexusAI/repo_mode.py context . "review app.py"
python NexusAI/repo_mode.py task . "generate a patch review"
python NexusAI/repo_mode.py test .
python NexusAI/repo_mode.py rollback .
```

## Limitations

- v0.1.1 does not apply patches automatically.
- v0.1.1 does not start v0.2 UI integration.
- v0.1.1 does not train or improve the model.
- Context selection is deterministic and heuristic-based; it may miss project-specific conventions.
- Test suggestions are recommendations only and must be run by a user or approved tool flow.

## Safety

- Generated directories and caches are ignored.
- Sensitive paths such as `.env`, credentials, secrets, private keys, `.git`, `node_modules`, `dist`, and `build` are blocked in patch review helpers.
- Dangerous command-like content is flagged.
- `coder_agent` always returns `auto_applied: false`.

## Version Boundaries

- v0.1: controlled local beta engine with routing, validation, repair, fallback, and release documentation.
- v0.1.1: deterministic coder tools for indexing, context, planning, tests, and risk review.
- v0.2: future UI TypeScript to Python bridge. Not included here.

## Next Steps

- Merge v0.1.1 after gates pass.
- Start v0.2 from the updated `main` only after v0.1.1 is merged.
- Keep v0.3 autonomy blocked until v0.1.1 and v0.2 are both merged and validated.
