# Nexus AI v0.1 Finalization Plan

Documento de controle para fechar uma v0.1 real, segura e usavel do Nexus AI.

## Objetivo da v0.1

A v0.1 deve ser um assistente local de desenvolvimento com aprovacao humana, capaz de:

- Ler contexto de projeto.
- Planejar uma alteracao.
- Propor patch revisavel.
- Validar codigo antes de entregar.
- Criar preview quando aplicavel.
- Bloquear comandos perigosos.
- Bloquear escrita fora do workspace.
- Bloquear fetch inseguro.
- Usar provider real quando configurado.
- Usar o modelo local somente dentro de rotas controladas.
- Gerar relatorio de benchmark e release reproduzivel.

## Fora do Escopo da v0.1

- Agente totalmente autonomo aplicando mudancas sem aprovacao.
- Execucao arbitraria de shell.
- Treino grande do zero.
- Suporte perfeito para todos os frameworks.
- Navegacao web aberta sem allowlist.
- Modelo local como fonte unica de verdade para tarefas complexas.

## Release Gates

### Modelo

- [ ] `instruction_following >= 0.30`
- [ ] `compile_like_rate >= 0.70`
- [ ] `repetition_score >= 0.90`
- [ ] `wrong_domain_rate <= 0.15`
- [ ] `syntax_error_rate <= 0.25`

### Tarefas Reais

- [ ] `50` tarefas reais executadas
- [ ] `resolved >= 30`
- [ ] `resolved + assisted >= 45`
- [ ] `dangerous_command_executed = 0`
- [ ] `write_outside_workspace = 0`
- [ ] `rollback_functioning = true`
- [ ] `replay_recoverable = true`

### Seguranca

- [x] Fetch URL exige HTTPS.
- [x] Fetch URL bloqueia localhost/IP privado.
- [x] Fetch URL bloqueia DNS resolvendo para IP privado.
- [x] Fetch URL revalida redirect.
- [x] Fetch URL limita redirects.
- [x] Fetch URL limita tamanho de resposta.
- [x] Fetch URL bloqueia content-type inseguro.
- [x] Command runner usa allowlist.
- [x] Command runner rejeita `cwd` fora do repositorio.
- [x] Command runner deduplica `runId`.
- [x] Command runner aplica timeout.
- [x] `write_file` exige `expectedHash`.
- [x] `patch_file` rejeita arquivo stale.
- [x] CSP bloqueia inline scripts em respostas do app.
- [x] UI real sem handlers inline detectaveis.
