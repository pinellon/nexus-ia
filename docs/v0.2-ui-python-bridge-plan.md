# Nexus AI v0.2 UI Python Bridge Plan

Status: implemented as a supervised beta bridge.

## Scope

v0.2 connects the TypeScript web UI to the controlled Python NexusAI runtime through a safe local bridge. It does not add autonomous patch application, model training, tokenizer work, cloud training, or v0.3 autonomy.

## API Contract

Request:

```json
{
  "mode": "task",
  "task": "review project structure",
  "suite": "smoke_25",
  "root": ".",
  "options": {
    "maxTaskSeconds": 60,
    "maxSuiteSeconds": 1200,
    "modelTimeoutSeconds": 30,
    "repairTimeoutSeconds": 20
  }
}
```

Supported modes:

- `task`
- `suite`
- `index`
- `plan`
- `coder-task`

Response:

```json
{
  "ok": true,
  "mode": "plan",
  "status": "success",
  "final_origin": "fallback",
  "provider_mode": "fallback",
  "task": "review project structure",
  "result": {},
  "metrics": {},
  "logs": [],
  "errors": [],
  "duration_ms": 1234,
  "auto_applied": false
}
```

`auto_applied` is always `false`. Any patch flow remains review-only and human-approved.

## Endpoints

- `GET /api/nexus/health`
- `POST /api/nexus/run`

Health includes:

- Python availability
- NexusAI runtime availability
- provider health from `/api/providers/health`
- supported modes
- `auto_apply_enabled: false`

## Bridge Safety

- Python is launched with `child_process.spawn`.
- `shell` is always `false`.
- Arguments are passed as an array.
- Modes are allowlisted.
- `root` must stay inside the repository.
- Suites are limited to `smoke_10`, `smoke_25`, and `smoke_50`.
- `benchmarks/smoke_25.json` is normalized to `smoke_25`.
- stdout/stderr are capped.
- process timeout kills the Python child.
- missing Python and missing NexusAI files return structured errors.

## UI

The existing Nexus AI side panel now has a compact Python runner block with:

- mode selector
- task input
- suite input
- Run and Cancel controls
- status metadata
- JSON result, metrics, logs, and errors

The UI is intentionally minimal and does not redesign the IDE.

## Validation

Required gates:

```powershell
npm run typecheck
npm run build
npm test
Get-ChildItem NexusAI -Filter *.py | ForEach-Object { python -m py_compile $_.FullName }
cd NexusAI
python -m unittest test_controlled_components.py
cd ..
python -m unittest tests/test_coder_components.py
```

CLI validation:

```powershell
python NexusAI/repo_mode.py index --root .
python NexusAI/repo_mode.py plan --task "add validation for dangerous commands"
python NexusAI/repo_mode.py coder-task --task "review project structure and suggest tests"
```

Smoke validation is only required if Python runner behavior changes. v0.2 does not change the Python runner.

## Explicit Non-Goals

- No v0.3 autonomy.
- No auto-apply of patches.
- No training or custom model work.
- No tokenizer or cloud training.
- No broad UI redesign.
