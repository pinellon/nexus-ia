# Nexus AI v0.1 Release Report

Date: 2026-06-08

## Official Positioning

Nexus AI v0.1 is a controlled local assistant for development workflows. It is not yet an autonomous coding model.

This release can be marked as a beta/local release because the controlled product layer is stable enough for local development workflows with human approval. The model itself remains experimental.

## General Status

Status: beta/local controlled release candidate.

The release is ready to be documented as v0.1 beta/local when the final local gates pass after these documentation and CI updates. It should not be presented as autonomous coding AI.

## What Works

- Node/web app builds and typechecks.
- Vitest suite passes.
- Python controlled components pass unit tests.
- Controlled task routing works in smoke reports.
- Output validation, repair, and fallback paths produce valid final artifacts.
- Reports include provider/fallback origin fields such as `final_origin`, `fallback_usage_rate`, and `model_direct_pass_rate`.
- Safety checks report zero dangerous commands executed without approval.
- Safety checks report zero writes outside the project.
- Smoke reports show no leftover Python processes.
- Provider health is available through `/api/providers/health`.

## What Does Not Work Yet

- The local model is not a reliable direct coding model.
- `model_direct_pass_rate` is `0.0` in the official smoke reports.
- `fallback_usage_rate` is `1.0`; deterministic fallback is the primary reliability path.
- Desktop packaging is not ready; `desktop:build` remains a placeholder.
- Rollback is not fully proven by the smoke criteria because the official smoke runs did not include a resolved `patch_rollback` task.
- The frontend folder is not part of this v0.1 release scope.

## Fallback Interpretation

`fallback_usage_rate: 1.0` is an intentional release disclosure, not a hidden success. The v0.1 product is reliable because the controlled layer routes, validates, repairs, and falls back deterministically. The direct model is not the reliability source.

This is acceptable for a beta/local controlled assistant. It is not acceptable as evidence that Nexus AI is an autonomous coding model.

## Direct Model Limitation

`model_direct_pass_rate: 0.0` means direct model output did not produce final semantically passing results in the official smoke reports. Future work must improve direct model generation separately from v0.1 release readiness.

The v0.1 release should describe the model as experimental.

## Rollback Status

Smoke criteria currently report:

- `rollback_functioning: False`

Reason: the metric is generated by checking whether any smoke task has `type == "patch_rollback"` and `status == "resolved"`. The latest official smoke reports did not include such a resolved task.

Rollback itself exists in repo mode:

- `NexusAI/patch_manager.py` implements `rollback_last`.
- `NexusAI/app.py` exposes `/repo/rollback`.
- `NexusAI/test_controlled_components.py` covers rollback through direct repo mode and API tests.

Release decision: rollback is partially covered and documented, but not counted as a fully passed smoke gate for v0.1. Add explicit smoke coverage for `patch_rollback` in v0.1.1 or v0.2.

## Provider Health

Provider health is now available at:

```text
GET /api/providers/health
```

Response shape:

```json
{
  "ok": true,
  "provider_mode": "real|mock|fallback|unavailable",
  "model_available": true,
  "fallback_available": true,
  "notes": []
}
```

The endpoint reports whether a direct model is available and whether the deterministic fallback path is available.

## Known Risks

- Users may overestimate the model if the release is marketed as autonomous AI.
- Fallback-heavy success can hide direct model weakness unless reports are read carefully.
- Desktop delivery is not ready.
- Rollback has unit/API coverage but lacks a passed official smoke gate.
- CI does not run heavy smoke suites by default.

## Release Decision

Decision: release as v0.1 beta/local controlled assistant.

Do not release as:

- autonomous coding model
- production autonomous agent
- desktop-ready packaged product
- evidence of strong local model quality
